# 美术风格指南

## 一、总方向

- **表现方式**：抽象的**格子 / 节点俯视 + 面板驱动**——地图、探索、事件、养成尽量用图标、色块、文本面板表达，少做场景与角色动画。
- **视觉皮肤**：**中式像素风**（2D / 2.5D 俯视）。
- **核心原则**：美术为短板，优先**省资源、可复用、内容易扩展**；把美术投入从"场景 / 角色逐帧动画"转移到"**UI、图标体系、关键点睛立绘**"上，让玩家专注于玩法与抉择。

---

## 二、各部分表现规格

| 部分 | 表现 | 省资源做法 |
| --- | --- | --- |
| **大地图** | 格子 / 节点俯视，区域 / 据点 / 驿站用像素**图标 + 色块** | 复用 tile；**侵蚀度用色调**表达（越近酆都越暗沉冷红） |
| **区域内** | 以面板 / 节点为主（不做大型可走场景） | 复用 tile / 图标 |
| **事件 / 剧情** | 文本 + 面板 + 像素**头像**；关键转折才上立绘 | 图文驱动，无动画过场（也契合"渐进揭示真相"叙事） |
| **战斗** | 1v1，双方各一个像素 sprite + 特效 + **数值飘字** | 低帧 / 半静态，**姿态切换 + 特效**替代逐帧大动作 |
| **NPC** | **部件化像素头像**（脸 / 发 / 服饰 / 配饰组合批量生成） | 关键角色才单独精绘立绘 |
| **UI** | 统一像素 UI、图标体系、面板排版 | 美术重心放这里，强化质感与辨识度 |
