# 时间与回合系统设计

## 一、时间单位

- 最小单位为「**天**」，累积为「**月**」。
- **月是结算周期**（每月固定天数，具体数值待定）。

---

## 二、时间流动规则

- **与现实时间无任何绑定**，玩家不行动时游戏时间**静止**。
- **仅"做事"消耗时间**：读书、研习、锻造、跨区域移动等，各消耗若干天。
- **同区域内移动 / 交互不消耗时间**（区域内自由活动）。
- 时间是核心资源之一；多数行动**同时消耗"精力（精）"**（见 `属性.md` · 精气神）。

---

## 三、行动与消耗（框架）

| 行动 | 时间 | 其他消耗 |
| --- | --- | --- |
| 读书 / 研习功法 / 研习技艺 | 若干天 | 精力（**不消耗神**） |
| 锻造 / 炼丹 / 制造 | 若干天 | 精力 + 材料 |
| 跨区域移动 | 视距离若干天 | 精力（可选） |
| 闭关修炼 | 可指定时长 | 精力 |
| 战斗 | 战斗内为半即时结算，不按"天"计 | 气血（承伤）/ 神（招式） |

> 具体天数与消耗数值由数值组在后续细化。

---

## 四、过月结算（月度统一计算）

跨月时一次性批量处理，避免逐日模拟：

1. **NPC 行动与状态**推进。
2. **事件生成 / 刷新**。
3. **区域侵蚀度变化**：被净化的侵蚀度逐步回升；按当前侵蚀度刷新区域事件与敌人等级。
4. **所有角色年龄 +**，并进行**寿元结算**（见下）。
5. **资源产出结算**（村庄等经营产出）。

---

## 五、寿命系统（统一寿元上限制）

> **一套规则覆盖所有角色（主角 + 全部 NPC）**，算法只有"过月加龄 → 到龄死 → 缺人补"。

- 每个角色都有 `年龄` 与 `寿元上限`。
- 过月时 `年龄 +`；当 `年龄 ≥ 寿元上限` → **自然死亡**。
- **寿元上限**受资质 / 境界 / 丹药影响：境界高者寿元略长，但**人界不长生**，延寿幅度有限。
- **死亡处理（统一规则，仅结果分支）**：
  - **主角死亡** → 触发**轮回重生**（依已收集神器碎片获加成；详见轮回成长线）。
  - **NPC 死亡** → 离场；若担任门派职位（如掌门），由门派现有成员顶替；门派**按需生成新弟子**补员（**无血缘繁衍 / 无逐日生老病死**）。

---

## 六、与其他系统的衔接

- **精气神（属性）**：行动消耗精力，战斗消耗气血 / 神。
- **核心循环 / 世界**：过月推进侵蚀度回升与事件刷新。
- **轮回成长线**：主角死亡的统一出口（轮回渐强）。
- **村庄经营**：可选产出在过月结算中计算（含远程运营）。
