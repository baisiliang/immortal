# 美术方向 · 决策记录

## 背景

- 美术是实现短板，目标：**最省美术资源**，且让玩家专注玩法。

## 已定

- **表现方式**：采用**抽象格子 / 节点俯视 + 面板驱动**的实现路子（地图 / 探索 / 事件 / 养成尽量用图标、色块、文本面板，少做场景与角色动画）。
- **视觉皮肤**：转化为**中式像素风**（2D / 2.5D 俯视）。
- **美术投入重心**：从"场景 / 角色逐帧动画"转移到 **UI、图标体系、关键点睛立绘**。
- **分部分规格**：大地图格子 + 图标（侵蚀度用色调）；区域内以面板 / 节点为主；事件图文 + 头像；战斗 1v1 简 sprite + 特效 + 飘字；NPC 部件化头像；关键角色才精绘立绘。

## 待定

- 像素分辨率 / 网格、调色板、帧数规范。
- 区域内是否做轻量可走小场景（暂定以面板 / 节点为主）。
