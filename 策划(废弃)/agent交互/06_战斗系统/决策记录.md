# 战斗系统 · 决策记录

## 已定

- **形态**：半即时回合制 1v1，时间持续流动、可暂停下令；核心 = 出招速度 + 式序列。
- **出招速度**：由基础属性、功法运转、装备搭配、武器性能综合决定，影响运转"式"快慢与抢先。
- **武技与式（核心）**：
  - 每个武技自带固定"式"序列（如 崩 / 拿 / 破）。
  - 运转武技 = 按序列逐个施展式；序列特定节点可施展武学招式（如第 N 个「破」施展第 N 式，第 3 个「破」直接最终式）。
  - 施展招式消耗对应的式；提前施展也消耗式。
  - 至少装备 1 个武技，起手有默认武技。
- **切换武学的式保留**：保留"已施展式序列最长后缀"中"匹配新武技式序列前缀"的那段作为新进度（见正式文档示例）。
- **资源**：精血（HP）、真气（蓝，招式 / 护体消耗、可调息）、护盾（护体耗真气生成、神魂/直伤无视）。
- **胜负**：气血归零为重伤而非必死，战败惩罚宜轻（轻松基调）。

## 待定

- 架势 / 破绽 / 拆招等防御博弈层是否引入。
- 运转中的防御 / 格挡机制。
- 敌方 AI、净化结算规则、各项数值。
- 时间节奏模型曾考虑"行动槽 / 速度条"，最终以"出招速度 + 式序列 + 时间轴"为准。
