# 时间与寿命 · 决策记录

## 已定

- **时间单位**：天 + 月；月为结算周期（每月天数待定）。
- **行动消耗**：做事消耗时间 + 精力（精）；研习**不消耗神**；战斗用气血 / 神。
- **过月结算**：NPC 行动、事件生成 / 刷新、区域侵蚀度回升与刷新、所有角色年龄 +、寿元结算、村庄产出结算。
- **寿命系统（统一寿元上限制，覆盖主角 + 全部 NPC）**：
  - 每个角色有 `年龄` + `寿元上限`；过月 `年龄+`，`年龄 ≥ 寿元上限` → 自然死亡。
  - 寿元上限受资质 / 境界 / 丹药影响；**人界不长生**，高境界仅略延寿。
  - 死亡处理：主角 → 轮回重生（依碎片获加成）；NPC → 离场，职位由现有成员顶替，门派按需生成新弟子补员（**无血缘繁衍、无逐日生老病死**）。
  - 算法仅"过月加龄 → 到龄死 → 缺人补"。

## 关键约束

- NPC 寿命机制必须**统一**（不分层、不特殊处理），且**简单**。
